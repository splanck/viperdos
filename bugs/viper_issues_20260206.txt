Executive Summary
-----------------
Analyzed 456+ source files (~115,000 SLOC) across kernel (~50K) and userspace
(~65K) components. This review builds on the January 29, 2026 report and
covers NEW findings only — issues already fixed in that report are not
repeated.

Identified 47 actionable issues across the following categories:

  - Code Duplication:            14 issues

                    PART 1: CODE DUPLICATION ISSUES
================================================================================

ISSUE #1: ViperFS On-Disk Structures Duplicated in Tools
  Files: tools/mkfs.viperfs.cpp (lines 37-155),
         tools/fsck.viperfs.cpp (lines 29-104)
  Type: duplication (HIGH)
  Description: Both tools independently define identical Superblock, Inode,
    DirEntry structs, type aliases (u8/u16/u32/u64), and filesystem constants
    (VIPERFS_MAGIC, VIPERFS_VERSION, BLOCK_SIZE, INODE_SIZE, etc.).
  Impact: ~150 lines duplicated; any spec change requires updating both files.
  Fix: Create shared tools/viperfs_format.h header.
  Status: [FIXED]

ISSUE #2: libc Syscall Declarations Duplicated Across 18+ Files
  Files: user/libc/src/unistd.c, stdio.c, stdlib.c, dirent.c, time.c,
         signal.c, stat.c, wait.c, poll.c, mman.c, utsname.c, resource.c,
         socket.c, netdb.c, spawn.c, random.c, pthread.c, ioctl.c
  Type: duplication (MEDIUM)
  Description: Each file independently declares extern prototypes for
    __syscall0 through __syscall6. Pattern repeats identically in every file.
  Impact: ~50-100 lines of duplicate declarations.
  Fix: Create user/libc/include/syscall_internal.h shared header.
  Status: [FIXED]

ISSUE #3: is_sys_path() Duplicated in vinit

                    PART 3: EFFICIENCY IMPROVEMENTS
================================================================================

ISSUE #18: O(n) Timer Lookup in timerwheel.cpp
  File: kernel/lib/timerwheel.cpp (lines ~63-69)
  Type: inefficiency (MEDIUM)
  Description: find_timer(u32 id) does linear scan through MAX_TIMERS (64)
    entries. Every cancel() and lookup triggers O(n) scan.
  Fix: Add timer ID index array for O(1) lookup.
  Status: [FIXED]

ISSUE #19: Hardcoded Kernel Text Bounds for Callback Validation
  File: kernel/lib/timerwheel.cpp (lines ~252-259)
  Type: inefficiency + fragility (MEDIUM)
  Description: Callback validation uses constexpr 0x40000000-0x40800000
    range. If kernel code section grows or relocates, valid callbacks are
    silently rejected.
  Fix: Use extern linker symbols _stext/_etext for dynamic bounds.
  Status: [FIXED]

ISSUE #20: ~100 Lines Debug Output in Scheduler Hot Path
  File: kernel/sched/scheduler.cpp
  Type: inefficiency (MEDIUM)
  Description: Extensive serial debug output for "displayd" task scattered
    across 3 blocks (~100 lines total) in the scheduling hot path. Runs
    unconditionally even in production.
  Fix: Gate behind #ifdef VIPER_SCHED_DEBUG.
  Status: [FIXED]

ISSUE #21: No Double-Free Detection in Buddy Allocator
  File: kernel/mm/buddy.cpp
  Type: safety (HIGH)
  Description: free_pages() does not check if address is already in a free
    list. Double-free silently corrupts allocator state.
  Fix: Add page state check before freeing.
  Status: [FIXED]

ISSUE #22: Undocumented/Unnamed VirtIO blk Timeouts
  File: kernel/drivers/virtio/blk.cpp
  Type: magic_number (LOW)
  Description: INTERRUPT_TIMEOUT = 100000, POLL_TIMEOUT = 10000000 have no
    comment explaining units or rationale.
  Fix: Document as iteration counts with estimated duration.
  Status: [FIXED]
